generics in collections
mainly to make our code safer, clearer, and easier to maintain.

1. Type Safety (Avoids Runtime Errors)

Without generics, collections can store any type of object, which can cause errors at runtime.

ArrayList list = new ArrayList();
list.add("Hello");
list.add(10);   // allowed

String s = (String) list.get(1); // Runtime error!

With generics:
ArrayList<String> list = new ArrayList<>();
list.add("Hello");
// list.add(10); // Compile-time error

2. No Need for Type Casting

Generics eliminate the need to manually cast objects when retrieving them.
Without generics :
   String s = (String) list.get(0);
With generics :
   String s = list.get(0);
Cleaner and easier to read.

3. Improves Code Readability

Generics clearly show what type of data a collection is supposed to hold.
List<Integer> numbers;
List<String> names;


4. Better Compile-Time Checking

The compiler enforces correct usage and helps catch mistakes early.
Map<Integer, String> map = new HashMap<>();
map.put(1, "Apple");
// map.put("Two", 2); // Compile-time error
